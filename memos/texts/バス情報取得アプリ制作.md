# バス情報取得アプリ制作記録

- 期間: 2025/04/19 - 2025/04/20
- 対応OS: Android
- IDE: Cursor
- AIモデル: Claude 3.7 Sonnet, Gemini 2.5 Pro
- 開発言語: TypeScript
- フレームワーク: Ionic, Angular
- データベース: SQLite

# 概要
個人開発モチベーションが出てきたので、AI内蔵IDEであるCursorを使って2日でどこまでアプリを作れるか試してみることにしました。
構想自体は前々からあったので、それを要件定義として起こす部分からスタートしました。

# 作成するアプリの概要
- リアルタイムのバス情報を取得できるAndroidアプリ
- 取得したバス情報はローカルDBに保存され、アプリ内で参照できる

# 基本方針
- 資料作成、設計、コーディング、テスト全てをCursorにて作業を行う
- 資料作成と設計はマークダウンにて管理する
- Auto runモードを利用する
  - ※ユーザの許可無くコードの書き換えやターミナルコマンドの実行をごりごり行っていく恐ろしいモードです。その代わりとても速いです。
  (ワークスペース外のファイルは操作しないよう設定しています）

![001](../images/001.png)

>　自動実行モードでは、設定した許可リストと拒否リストに基づいてコマンドが実行されます。外部ソースからの潜在的なプロンプトインジェクションのリスクに注意し、自己責任でご利用ください。

時間が足りないので使えるものは使っていく方針です。Continueします。

```
# バス時刻取得アプリ要件定義

# 背景

- 広島県のバスは渋滞状況によって到着時刻が大きく異なるので、いつ到着するかの目処が付けづらい
- リアルタイムでバス情報を確認できるサイトとして、「くるけん」が存在するが、下記理由からやや不便
    - ブラウザを起動する必要があり、忙しい時は手間に感じる

    - 表示される到着予定時刻と実際の到着時刻が大きく異なる時がある

# ニーズ

- 日ごとに到着時刻を記録し、それを元にバス停にいつたどり着ければ間に合うかの指標になるようなデータを表示できるものが欲しい
    - 例えば「以前このバス停に〇〇分に付いているからおそらく到着時刻は〇〇分だろう」「〇〇分についておけばほぼほぼ間に合うだろう」など判断したい

```

```
### 説明
- ユーザーがアプリ起動後に最初に見る画面です。

### 表示情報
- リアルタイムバス情報リスト
  - 説明
    - 縦にスクロールできるリストです。
    - リアルタイムバス情報が表示されます。
    - 一度に取得される情報には上限があります。
    - リストの一番上にスクロールされた状態で、さらに上にスクロールされるとリアルタイムバス情報が更新されます。
    - リストの一番下までスクロールされると、自動で情報の取得が行われます。
  - リアルタイムバス情報コンポーネント
    - バスの到着予定時刻（もしくはバスの平均到着時刻 / 設定で変更可能）
    - バスがあと何分後に到着するか
    - バスの定刻
    - バスの終点
    - バスの運行状況
    - バスの混雑度
    - バスの運行会社
    - バスの路線番号
    - バスの通過バス停

- 現在の指定バス停ラベル
  - 説明
    - 現在リアルタイムバス情報と過去の到着時刻リストに表示するバス停が表示されるラベルです。

- お気に入りバス停リスト
  - 説明
    - リアルタイムバス情報リストと過去の到着時刻リストに表示するバス停を指定できるリストです。
    - お気に入りに登録したバス停が表示されます。
    - 横にスクロールできます。
    - 検索アイコン押下時に検索欄に置き換わります。
  - バス停名コンポーネント
    - バス停名
    - 押下時処理
      - 押下されたバス停名を現在の指定バス停ラベルに表示します。
      - 指定されたバス停名コンポーネントは色が変わります。

- 検索欄
  - 説明
    - バス停名や路線名を検索できる欄です。
    - 通常時は表示されず、検索アイコンが押下された時にお気に入りバス停リストに置き換わる形で表示されます。
    - 検索欄が表示されている時に検索欄以外の部分がタップされると検索欄の表示が消え、お気に入りバス停リストが表示されます。
    - 入力された内容をDBに保持しているバス停名から検索します。
    - 検索結果はプルダウンとして表示されます。
    - 検索は検索欄の入力が一定時間（0.25秒程度を想定）行われていない場合に検索が自動で実行されます。
  - 検索結果プルダウン
    - 説明
      - 検索が実行されたあと検索欄の下に自動で表示されます。
      - 検索結果が多い場合縦にスクロールが可能です。
    - プルダウン押下時処理
      - 押下されたプルダウンのバス停を現在の指定バス停ラベルに表示します。

- 検索アイコン
  - 説明
    - お気に入りバス停リストの真左に配置されます。
  - 押下時処理
    - お気に入りバス停リストを検索欄に置き換えます。

- 過去のバス情報テーブル
  - 説明
    - 過去の到着時刻テーブルの真左に表示されます。
    - 過去の到着時刻テーブルの値がどのルートの情報なのかを表すためのテーブルです。
  - フィールド
    - 定刻
      - バスの定刻が表示されます。
    - 平均到着時刻
      - 現在表示されている過去の到着時刻テーブルの情報からバスの平均到着時刻を計算した値が表示されます。

- 過去の到着時刻テーブル
  - 説明
    - 過去のバス情報テーブルの真右に表示されます。
    - 過去のバスの到着時刻などの情報を表示します。
    - フィールド名は日付となっており、右から順に今日の日付、今日の日付-1日、今日の日付-2日...と続きます。
    - 一度に取得される日付には限度が存在します。
    - 一番左までスクロールした状態で左にスクロールをすることで更に過去の日付の情報を取得することができます
  - フィールド
    - 到着時刻フィールド
      - 実際に表示されるフィールド名は日付となります。
      - フィールド名の日付のフォーマットは[mm/dd]となります。
      - 内容にバスの到着時刻, 遅延時間, 混雑率が表示されます。
```
